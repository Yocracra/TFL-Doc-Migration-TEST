# 1\.  Access Regulation & Credentials

# 1.1  Access Points.

For IOT related projects e.g. microcontrollers, you should only be connected to the TamuFabLab_STUDENT wireless network. Ask Lab Staff for connection information.

Any project requiring usage of the lab server’s computing resources will require communication with Lab Staff[[a]](<#cmnt1>).

Note: Completion of the network assignment is mandatory prior to network access. The code for the assignment can be found in the [Assignment Page.](<https://www.google.com/url?q=https://docs.google.com/document/d/1aypmYK76DssTU0G9nlWYycDx4vtaeWZQvOHfrLW1A7w/edit?tab%3Dt.0&sa=D&source=editors&ust=1776804236110867&usg=AOvVaw0Hl19-E0evK03XoyUkkHEe>) Once the assignment is completed, email the Lab Staff via email to finalize your credentials. 

        

# 1.2 Credential Management

For IOT/ Microcontroller Projects ensure the TamuFabLab_STUDENTS network password is securely held in a file that is included in the project .gitignore, personal projects will require demonstration to ensure credentials are properly handled.

For Server access, the user is expected to be able to use SSH and SSH Keys, the user will be given a set of SSH keys as needed for their project if approved.

For Personal Devices, the network password is not to be saved during or after connection to the TamuFabLab_STUDENTS wireless network on personal devices. 

# 2\. Hardware & Device Security

# 2.1 Security Guidelines

It is expected that users of the TamuFabLab_STUDENT network are familiar with basic network security practices (e.g. firewalling connected devices).

There is no expectation of privacy when using any Fab Lab network.

# 2.2 Approved Device List

Refer to section 5.4 of the [Operations Manual.](<https://www.google.com/url?q=https://docs.google.com/document/u/0/d/1eglXddWRoWSFrGuvTAB6qP-PFFrUNKDfj2g9WPZHaFw/edit&sa=D&source=editors&ust=1776804236113619&usg=AOvVaw0FrSDFcXTylAzuVb2ixLLN>) 

# 2.3 Prohibited Device List

Any device that utilizes a nonstandard WiFi connection or otherwise poses a risk to the network signal integrity or security is forbidden.

# 2.4 Prohibited Service/Use List:

  * Anything illegal.
  * Any Service that disrupts network quality or security.
  * Passwordless personal devices.
  * Any service that extends the network beyond the TamuFabLab_STUDENT SSID.
  * Any service that violates any Local/State/Federal Law or otherwise violates TAMU Policies or TAMU Network Policies.



# 3\.  Incident Response & Monitoring

3.1 IDS/IPS Monitoring:

All Tamu Fab Lab Prototyping Studio networks are monitored by UniFi IDS/IPS to detect and stop any suspicious network traffic. If network access is randomly lost or suspended, it is likely due to an IDS/IPS action, if you are experiencing network interruptions, please contact a staff member.

3.2 Network Monitoring:

All network devices on the TamuFabLab_STUDENT and TamuFabLab_MACHINES will have their network traffic logged. There is no expectation of privacy while using the network.

If you have questions or need assistance at any point, ask a Fab Lab staff member. Staff are always present during operating hours.
